# Project Recovery Instructions

This project is maintained in small, resumable checkpoints because the PC may shut down suddenly.

Before coding in a new session:

1. Inspect the project structure.
2. Read `AGENTS.md`, `PROJECT.md`, `CURRENT_TASK.md`, `CHANGELOG.md`, `NEXT_STEPS.md`, and `README.md`.
3. Run `git status --short --branch`.
4. Treat all uncommitted and untracked files as user work unless proven otherwise.
5. Continue from `CURRENT_TASK.md` and `NEXT_STEPS.md`; do not restart from scratch.

Working rules:

- Make small production-ready changes.
- Prefer existing project patterns and files.
- Do not rewrite the stack or remove working features without explicit approval.
- Update `CHANGELOG.md`, `NEXT_STEPS.md`, and `CURRENT_TASK.md` after meaningful work.
- Run the safest available checks after changes and record the result.
